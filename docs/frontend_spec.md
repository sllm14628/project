# 프론트엔드 화면 정의서 (UI Specification)

본 문서는 sLLM RAG 시스템의 프론트엔드(React) 화면 구성 및 주요 UI 컴포넌트를 정의합니다.

## 1. 전체 레이아웃 (Global Layout)
모든 화면은 기본적으로 좌측 사이드바와 우측 메인 콘텐츠 영역으로 나뉩니다.
* **사이드바 (Sidebar)**
  * **로고 및 프로젝트명**: 상단 배치
  * **네비게이션 메뉴**:
    * 💬 대시보드 (채팅/비교)
    * 📝 템플릿 관리 (Admin)
    * 📄 문서 관리 (Admin)
    * ⚙️ 모델 설정 (Admin)
  * **상태 표시**: 하단에 현재 연결된 서버 상태 또는 사용자 프로필 표시

## 2. 화면별 상세 구성

### 2.1 대시보드 화면 (다중 모델 비교 채팅)
사용자가 질문을 입력하고 여러 sLLM의 응답을 한눈에 비교하는 메인 화면입니다.
* **상단 바 (Header)**
  * 비교할 모델 선택 드롭다운 (최대 2~3개 모델 동시 선택 가능)
  * 적용할 프롬프트 템플릿 선택 드롭다운
* **메인 채팅 영역 (다중 컬럼 레이아웃)**
  * 선택한 모델 수에 따라 화면이 N분할(Split) 됨.
  * 각 컬럼의 상단에 **모델명** 표시.
  * 스크롤 동기화: 한 쪽의 채팅을 스크롤하면 다른 쪽도 같이 스크롤되어 동일한 질문에 대한 답을 비교하기 쉽게 구성.
* **입력 영역 (Bottom Input)**
  * 넓은 텍스트 입력창 (Shift+Enter 줄바꿈 지원)
  * RAG 토글 스위치: 업로드된 문서를 기반으로 답변할지 일반 대화할지 선택
  * 전송 버튼

### 2.2 템플릿 관리 화면 (Admin)
RAG 성능을 높이기 위한 System Prompt 템플릿을 관리하는 화면입니다.
* **목록 영역 (List View)**
  * 등록된 템플릿 목록 테이블 (템플릿명, 설명, 최종수정일)
  * '새 템플릿 생성' 버튼
* **상세/수정 영역 (Detail/Edit View - 모달 또는 우측 슬라이드)**
  * 템플릿명 입력 필드
  * 프롬프트 내용 입력 필드 (Textarea)
    * *Tip*: `{context}`, `{question}` 과 같은 변수를 하이라이팅하여 표시.
  * 저장 / 삭제 버튼

### 2.3 문서 관리 화면 (Admin)
RAG 검색에 사용될 외부 지식 데이터(문서)를 업로드하고 관리하는 화면입니다.
* **업로드 영역 (Drag & Drop Zone)**
  * PDF, TXT 파일 드래그 앤 드롭 업로드 지원
  * 업로드 진행률 프로그레스 바 표시
* **문서 목록 테이블**
  * 파일명, 크기, 업로드 일시, 상태(임베딩 완료/진행중/실패)
  * 삭제(휴지통) 버튼

### 2.4 모델 설정 화면 (Admin)
비교할 대상이 되는 로컬 또는 API 기반 sLLM 목록을 관리합니다.
* **모델 목록 리스트/카드 뷰**
  * 모델명 (예: Llama-3-8b, Mistral-v0.3 등)
  * 활성화/비활성화 토글 스위치 (대시보드 노출 여부)
  * API Endpoint URL 설정 필드 (로컬 Ollama 주소 등)
  * 연결 테스트 (Ping) 버튼

## 3. 공통 UI 컴포넌트 (Components)
* `MessageBubble`: 사용자 질문과 AI 답변을 표시하는 말풍선. Markdown 렌더링 지원 (코드 블록, 표 등).
* `DropdownSelect`: 모델 및 템플릿 선택용 커스텀 셀렉트 박스.
* `DataTable`: 관리자 화면에서 목록을 보여주기 위한 정렬/페이징 지원 테이블.
* `ToggleSwitch`: RAG 활성화 및 모델 활성화를 위한 스위치.
* `ToastNotification`: 저장 성공, 업로드 완료, 에러 메시지 등을 띄우는 알림 팝업.
